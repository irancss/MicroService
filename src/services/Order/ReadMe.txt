3. میکروسرویس سفارشات (Order Service)
وظایف:
ثبت و اعتبارسنجی سفارشات (چک کردن موجودی و پرداخت).
مدیریت چرخه حیات سفارش (لغو، بازگشت، تغییر آدرس).
پیاده‌سازی الگوی Saga (Choreography یا Orchestration) برای تراکنش‌های توزیع‌شده.
همگام‌سازی با انبار، پرداخت، و تحویل.
گزارش‌گیری لحظه‌ای از وضعیت سفارشات.
دیتابیس پیشنهادی:
PostgreSQL: برای یکپارچگی داده‌ها و پشتیبانی از تراکنش‌های پیچیده.
زبان/فریمورک:
.NET Core: با Entity Framework Core برای ORM قوی.
جایگزین پیشنهادی: Java + Spring Boot با Axon Framework برای مدیریت Saga.
تکنولوژی‌های کمکی:
Apache Kafka: برای ارسال رویدادهای سفارش به سرویس‌ها.
Dapr: برای ساده‌سازی مدیریت تراکنش‌های توزیع‌شده.
